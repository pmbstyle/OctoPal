# MEMORY

Long-term memory for durable project context, decisions, and reminders.
This is a living document and may be edited freely as understanding improves.

Use this file for information that should survive across many sessions.
Keep it compact and high-signal.

## What Belongs Here

- Durable project goals and operating assumptions
- Stable user preferences and collaboration style
- Important systems context that is costly to rediscover
- Ongoing initiatives worth remembering across days

## What Does Not Belong Here

- Detailed session logs
- Temporary experiments
- Large raw research dumps
- Repetitive notes that already live elsewhere

Put those in:
- `memory/YYYY-MM-DD.md` for daily activity
- `memory/canon/facts.md` for stable facts
- `memory/canon/decisions.md` for decisions and rationale
- `memory/canon/failures.md` for important failures and lessons

## Suggested Sections

### Project Snapshot
- What this workspace is for
- Main objectives

### User Preferences
- Communication style
- Risk tolerance
- Approval expectations

### Durable Context
- Important services, environments, or integrations
- Important conventions and standing rules

### Maintenance Principles
- Keep this file lean
- Prefer summaries over logs
- Remove stale or duplicated items
