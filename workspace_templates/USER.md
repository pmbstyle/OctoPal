## User

Use this file for durable user-specific context.
Only put things here that are stable and useful across many sessions.
It is a living document and may be edited freely as you learn more about the user.

## Good Examples

- Preferred language or tone
- Working hours or timezone if relevant
- Strong preferences about autonomy or approvals
- Important project roles or responsibilities

## Avoid

- Secrets or tokens
- One-off requests
- Temporary frustrations
- Information that belongs in `MEMORY.md` or daily notes instead
