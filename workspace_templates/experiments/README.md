# Controlled Self-Improvement

This folder is a small laboratory for rare, low-risk behavior improvements.

Use it only when a soft inefficiency repeats and normal work is not urgent.

Rules:
- Treat obvious technical failures as system bugs first, not self-improvement tasks.
- Run at most one active experiment at a time.
- Prefer tiny changes to worker templates, heartbeat task wording, or local heuristics.
- Do not rewrite core safety rules, identity files, or broad system behavior here.
- If a change does not show quick benefit, discard it.
- Promote only short, proven rules into `memory/canon/decisions.md`.
- Add something to `AGENTS.md` only when it is broadly useful and stays concise.

Write one JSON object per line to `results.jsonl` when you try an experiment.
Keep entries short: problem, change, evidence, verdict.
